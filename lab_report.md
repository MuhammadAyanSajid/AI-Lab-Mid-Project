## Introduction
The chosen domain for this lab is the Maze Solver, which is a classical AI search problem. A maze can be modeled as a state-space search task where each cell in a 2D grid represents a state. The agent begins from an initial state, tries available actions, and searches for a goal state while avoiding obstacles.

In this project, `0` represents an open cell and `1` represents a blocked cell. The solver explores the maze using four search strategies: Breadth-First Search, Depth-First Search, Iterative Deepening Search, and A* Search. This domain is suitable for AI analysis because it clearly shows the behavior of uninformed and informed search algorithms.

## State Space Representation
The maze is represented as a 2D grid stored in Python. Each state is defined as a coordinate pair `(row, col)`.

- **Initial State:** The starting position of the agent in the maze.
- **Goal State:** The target position that the agent must reach.
- **State Space:** All valid grid cells that are inside the maze and are not blocked by walls.
- **Actions / Moves:** Up, Down, Left, and Right.

In the code, the `MazeSolver` class checks whether a state is inside the grid and whether it is open before allowing movement to that state. This ensures that only legal transitions are explored.

## Methodology
The search algorithms were implemented in a strictly object-oriented way inside the `MazeSolver` class.

### Breadth-First Search (BFS)
BFS was implemented using `collections.deque` as a FIFO queue. The algorithm explores nodes level by level, which makes it complete and optimal for unit step cost problems like this maze.

### Depth-First Search (DFS)
DFS was implemented using a Python list as a LIFO stack. It explores as deeply as possible along one branch before backtracking. This can find a solution quickly in some cases, but it is not guaranteed to return the shortest path.

### Iterative Deepening Search (IDS)
IDS was implemented by repeatedly applying Depth-Limited Search with increasing depth limits. It combines the low memory usage of DFS with the completeness of BFS.

### A* Search
A* was implemented using `heapq` as a priority queue. Each node is ordered by `f(n) = g(n) + h(n)`, where `g(n)` is the path cost from the start and `h(n)` is the heuristic estimate to the goal. This satisfies CLO 1 because the solution uses the correct data structures: Queue for BFS, Stack for DFS, and Priority Queue for A*.

## Heuristic Evaluation
The heuristic used for A* is **Manhattan Distance**:

$$
h(n) = |r_1 - r_2| + |c_1 - c_2|
$$

where `(r_1, c_1)` is the current cell and `(r_2, c_2)` is the goal cell.

This heuristic is admissible because the maze allows movement only in four directions and each move has a unit cost. Manhattan Distance never overestimates the actual shortest path cost in such a grid, so A* remains optimal.

## Analysis & Conclusion
The results show clear behavioral differences between uninformed and informed search.

BFS and A* both found the shortest path with a cost of 11. BFS achieved optimality because it expands states level by level, but it explored more nodes than A*. A* was more efficient because the Manhattan heuristic directed the search toward the goal and reduced unnecessary expansion.

DFS explored fewer nodes than BFS in this run, but it produced a longer path with cost 15. This shows the weakness of DFS in pathfinding problems: it can find a solution quickly, but that solution is not guaranteed to be optimal.

IDS also found the optimal path with cost 11, which matches BFS and A*. However, it explored the largest number of nodes because it repeats depth-limited searches from scratch at each depth level. This makes IDS complete and memory-efficient, but less efficient in time than BFS or A*.

From a complexity viewpoint, BFS and IDS require more memory than DFS in general because they maintain larger frontiers or repeated exploration states. A* uses additional memory for the priority queue and cost tracking, but in return it often achieves a better balance between completeness, optimality, and efficiency. In this maze, A* performed best overall because it explored the fewest nodes while still returning the optimal path.

Overall, this lab demonstrates that algorithm choice strongly affects search performance. Uninformed methods are useful for understanding the search process, while informed search, especially A*, is more suitable when a good heuristic is available.
